# Experiment

## Overview

This experiment tracks how chord transition performance changes across structured practice sessions over time.

Each session follows the same routine, allowing performance to be compared across time.

---

## Reference

Practice routine adapted from:
https://www.youtube.com/watch?v=WtXb90taPOY

---

## What is a Session?

A session is one complete run of the practice routine.

In each session, the same set of conditions is repeated:
- 4 difficulty modes
- 6 BPM levels (60–110)

This creates a fixed grid that stays identical across sessions.

---

## Practice Flow

At each BPM, the player alternates between A and E chords using a 4-beat cycle.

Each condition (mode + BPM) is repeated for 4 trials.

A trial is counted as successful if:
- the chord change is clean  
- the timing is correct  

---

## Modes (Difficulty)

Modes control how much time is available to switch chords:

- Mode 1 → most time to switch  
- Mode 2 → less time  
- Mode 3 → very limited time  
- Mode 4 → minimal time (hardest)

As mode increases, the gap available to change chords becomes smaller.

---

## BPM (Speed)

BPM controls how fast each cycle runs:

60 → slowest  
110 → fastest  

Higher BPM means less time per beat, increasing difficulty.

---

## Structure of a Session

Each session contains:

- 4 modes  
- 6 BPM levels  
- 4 trials per condition  

Total:

4 × 6 = 24 blocks per session  
Each block = 4 trials

---

## Measurement

Each block produces:

- successful_trials (0–4)
- clean_ratio = successful_trials / 4

This captures how consistently clean the chord transitions are under that condition.

---

## Why this design?

The structure keeps everything constant except:
- speed (BPM)
- available transition time (mode)

This makes it possible to observe how performance changes under controlled conditions.
